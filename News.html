Intune Weekly Update

New Features

Enhanced Device Compliance Policies: Administrators can now define more specific compliance rules, which allow for stricter security enforcement. This ensures that only devices meeting these granular requirements can access corporate resources.

Advanced Reporting Dashboards: A newly designed dashboard provides IT administrators with in-depth analytics about device health, policy compliance, and security threats. These insights enable proactive problem-solving and optimization of endpoint management strategies.

Bug Fixes

Resolved Policy Deployment Delays: Some users reported that policies were taking an unusually long time to be pushed to devices. This issue has now been fixed, ensuring policies deploy more efficiently and without unnecessary delays.

Fixed Application Installation Errors: A problem affecting certain Android devices, where apps failed to install due to inconsistent configurations, has been resolved. This fix ensures a smoother deployment process for mobile applications across diverse devices.

Known Issues

iOS Profile Installation Prompts: Users installing configuration profiles on iOS devices might experience repeated prompts asking for installation confirmation. Microsoft is currently investigating the root cause and working on a resolution.

Windows Autopilot Enrollment Hangs: Some Windows devices may freeze during the Autopilot enrollment process. A temporary workaround involves restarting the device and retrying enrollment, though a permanent fix is being developed.

Additional Notes

Upcoming Deprecation: Support for Windows 8.1 devices will officially end on July 1, 2025. Organizations using these devices are advised to upgrade to a newer, supported version of Windows to maintain security and compatibility.

Feedback Opportunity: Microsoft is seeking feedback from administrators on the newly introduced reporting dashboards. Users are encouraged to participate in a short survey to help improve the usability and effectiveness of the analytics platform.
